Golang学习之路—容器（数组、切片、映射）

Go基础知识
Golang

一、数组（Array）

  • 数组是由特定类型元素组成的有序序列，Go的数组长度在声明后是固定的，不能改变，
  • 数组的长度也是数组类型的一部份，[3]int 与 [4]int 是不同的类型。
0x00 数组声明

var arrayName [len]T

  • len为数组长度，T为元素类型。
0x01 数组初始化

  • 第一种

    var test [3]int      //[0 0 0]。默认元素都被初始化为0。
var test = [3]int{}  //与上面相同。
var test = [3]int{4, 5, 6}

  • 第二种：索引赋值

    var test [3]int

test[0] = 1
test[1] = 3

  • 第三种：自动确定长度，推荐使用。

    test := [...]int{1, 2, 3}       //长度使用 … 时，会根据后面的初始化的元素个数确定长度。
test := [...]int{5: 5, 9: 1}    //映射赋值，下标为5的元素值为5，下标为9的元素值为1，数组长度为10。
0x02 数组操作

  • 遍历数组

    test := [...]int{5: 5, 9: 1}

for i, v := range test {
        fmt.Println(i, v)   //i为下标，v为值
}

  • cap()与len()

    • len() 函数可以获取数组和切片的元素个数，即数组长度；cap()函数可以获取数组和切片的容量。
    • 在数组中由于长度固定不可变，因此len()和cap()的输出永远相同。

  • 数组是值类型，赋值和传参会复制整个数组。

    func main() {
        arr1 := [4]int{1, 2, 3, 4}
        arr2 := arr1
        arr1[2] = 5

        fmt.Println(arr1) //[1 2 5 4]
        fmt.Println(arr2) //[1 2 3 4]
}

  • 多维数组

    test := [4][2]int{{1, 2}, {3, 4}, {5, 6}, {6, 7}} //长度为4，类型为 [2]int 的二维数组
fmt.Println(test[0][0], test[1][0])

二、切片(slice)

  • 切片是引用类型，切片不定长。

  • 切片（slice）本质上是对一个数组的连续片段的引用。

0x00 切片的本质

本质上是一个对底层数组的引用，切片的结构包含：指针（底层数组首个被引用元素地址）、切片的长度、切片的容量。

0x01 切片声明及初始化

  • 第一种

    var s1 []int

s1 = []int{1, 2, 3}
s2 := []string{"a", "b", "c"}
    • 本质上先分别生成两个数组[3]int{1,2,3}、[3]string{"a","b","c"}，然后让s1、s2切片分别对其引用。

  • 第二种：直接引用数组片段。

    arr := [...]int{0, 1, 2, 3, 4, 5, 6, 7}

var s3 []int
s3 = arr[3:]     //[3 4 5 6 7]
s4 := arr[2:5]   //[2 3 4]

  • 第三种：使用make函数构造切片。

    • make( []T, size, cap ) //T为元素的类型；size为切片的长度；cap为容量，但是不影响后续增加且可省略。

      a := make([]int, 2)      //[0 0] 分配两个元素长度
b := make([]int, 3, 10)  //[0 0 0] 分配两个元素长度，但内部实际分配的10个空间，只是实际用3个。

    • 使用make函数生成的切片一定发生了内存分配，内存分配一定会进行初始化（用对应零值），即永远不会为nil。

      func main() {
    s1 := make([]int, 3, 10)
    fmt.Printf("%v、%d、%d", s1, len(s1), cap(s1)) //[0 0 0]、3、10

    var s2 []int

    fmt.Println(s1 == nil) //false
    fmt.Println(s2 == nil) //true
}
      func main() {
    s1 := make([]int, 5, 10)   //[0 0 0 0 0]。make会进行内存分配，内存分配会使用零值初始化
    for i := 0; i < 5; i++ {
        s1 = append(s1, i)
    }
    fmt.Println(s1) //[0 0 0 0 0 0 1 2 3 4]
}
0x03 切片的长度（len）与容量（cap）
  • 切片的长度可用len()函数，长度即元素个数。
  • 切片的容量用cap()函数,容量是切片引用的底层的数组从第一个引用元素到数组最后一个元素（注：不是引用的最后一个元素）的个数。

  • 代码示例

    // 代码示例1：
func main() {
    arr := [...]int{0, 1, 2, 3, 4, 5, 6, 7}

    s3 := arr[3:]   //[3 4 5 6 7]
    s4 := arr[2:5]  //[2 3 4]

    fmt.Println(len(s3), cap(s3))  //5 5
    fmt.Println(len(s4), cap(s4))  //3 6。s4是从arr数组的第2个元素开始引用，因此容量为 [2:],共6个。
}
0x04 切片特性

  • 切片是对底层数组的引用，因此底层数组值发生改变时，引用的值也会改变。

    // 代码示例1：
func main() {

    arr := [...]int{0, 1, 2, 3, 4, 5, 6, 7}

    s3 := arr[3:]     //[3 4 5 6 7]
    s4 := arr[2:5]    //[2 3 4]

    arr[3] = 3333333 //改变底层数组的值

    //切片也发生了改变
    fmt.Println(s3)   //[3333333 4 5 6 7]
    fmt.Println(s4)   //[2 3333333 4]

}

  • 修改切片的值，实际上是改变底层数组的值。所以所有的引用都会发生改变。

    // 代码示例1：
func main() {
    s1 := make([]int, 3, 10)
    s2 := s1
    s1[1] = 1 //实际上是改变底层数组

    fmt.Println(s1) //[0 1 0]
    fmt.Println(s2) //[0 1 0]
}
    // 代码示例2：
func main() {
    arr := [...]int{0, 1, 2, 3, 4, 5, 6, 7}
    s3 := arr[3:] //[3 4 5 6 7]
    s4 := arr[2:5] //[2 3 4]

    s4[1] = 111 //本质上是改变了底层数组arr[3]的值

    fmt.Println(arr) //[0 1 2 111 4 5 6 7]
    fmt.Println(s3)  //[111 4 5 6 7]
    fmt.Println(s4)  //[2 111 4]
}

  • 切片是引用类型，所以赋值是引用。copy()是拷贝，拷贝了底层数组。

    // 代码示例：
func main() {

    arr := [...]int{0, 1, 2, 3}

    s1 := arr[:]
    s2 := s1     //赋值
    s3 := make([]int, 4, 4)

    copy(s3, s1)  //copy

    arr[1] = 1111 //修改原数组

    fmt.Println(s1) //[0 1111 2 3]
    fmt.Println(s2) //[0 1111 2 3]，切片赋值是引用，所以底层数组改变也被改变了。
    fmt.Println(s3) //[0 1 2 3]，copy的没有因原数组改变而改变，因为copy的是底层数组。
}
0x05 切片操作

  • 元素追加：append()

    //基本操作
test := []int{}
            
test = append(test, 1)     //后面追加一个元素
test = append(test, 2, 3)  //后面追加两个元素
test = append(test, []int{4, 5, 6}...)  //后面追加三个元素，切片会被解包，…是解包符。

test = append([]int{0}, test...)  //前面追加一个元素

test := []int{1, 2, 4, 5}
test = append(test[:2], append([]int{3}, test[2:]...)...)  //中间插入元素3,中间插入只能采用间接的方式。

    • 当切片的容量还足够时，append()是修改底层数组。

      func main() {
    arr := [...]int{0, 1, 2, 3, 4}
    s1 := arr[:3] //[0 1 2]

    s1 = append(s1, 5) //[0 1 2 5]
    fmt.Println(arr)   //[0 1 2 5 4],本质是底层数组被修改
}
      func main() {
    arr := [...]int{0, 1, 2, 3, 4}
    s1 := arr[:3]  //[0 1 2]
    s2 := arr[3:]  //[3 4]

    s1 = append(s1, 5) //[0 1 2 5]

    fmt.Println(arr) //[0 1 2 5 4],本质是底层数组被修改
    fmt.Println(s2)  //[5 4]
}

    • 在切片的容量不足时，切片会进行扩容，即生成一个新的底层数组。

      func main() {
    arr := [...]int{0, 1, 2, 3, 4}

    s1 := arr[:]
    fmt.Println(len(s1), cap(s1)) //5 5

    s1 = append(s1, 5)
    fmt.Println(len(s1), cap(s1)) //6 10。进行了扩容，容量为原来的2倍

    arr[4] = 4444 //修改原数组，[0 1 2 3 4444]
    fmt.Println(s1) //[0 1 2 3 4 5]。原数组的修改不再影响切片，因为此时切片引用的不再是原数组。
}

    • 切片只能是数组的连续片段进行引用，不能断开。

      func main() {
    arr := [...]int{0, 1, 3, 4}

    s1 := append(arr[:2], arr[3:]...)  //[0,1,4]

    fmt.Println(arr)  //[0 1 4 4]，底层数组被修改覆盖
}

    • 几点总结：

      1. 每个切片会指向一个底层数组，这个数组能容纳一定数量的元素。当底层数组不能容纳新增的元素时，切片就会自动按照一定的策略进行“扩容”（前一次容量的2倍），此时该切片指向的底层数组就会更换，即产生新的底层数组，因此当发生扩容时，切片的地址也会改变（新的底层数组地址改变，切片指向的地址自然跟随新数组改变）。
      2. append（）函数的返回值是更新后的切片，用返回值赋值给切片变量实现更改切片的操作，之所以要用返回的切片对原切片进行更新，而不像Python中列表那样的原处修改，是因为更新的切片地址、容量可能发生改变了（扩容产生了新数组）。
      3. 在前面追加元素都会发生切片地址改变，因此开销要比在后面追加大。

  • 切片元素删除

    • 要从切片a中删除索引为index的元素，操作方法是 a = append(a[:index], a[index+1:]...)

  • 遍历切片

    test := []string{"Aqua", "Mikasa", "Yuihama", "Touka", "Tosaki"}

for i, v := range test {
    fmt.Printf("Index：%d ; Value：%s \n", i, v)
}

  • 切片排序：

    func main() {
    s1 := []int{2, 5, 1, 4, 0, 6, 8, 7}
    sort.Ints(s1)

    s2 := []string{"d", "b", "a", "c"}
    sort.Strings(s2)

    fmt.Println(s1)  //[0 1 2 4 5 6 7 8]
    fmt.Println(s2)  //[a b c d]
}
0x06 多维切片
test := [][]string{{"Aqua"}, {"Mikasa", "Yuihama"}}
test[0] = append(test[0], "Tosaki")
                
fmt.Println(test[0][0], test[0][1], test[1][1])

三、映射（Map）

  • Go语言的map是一种以键值对方式存储的无序集合数据结构，类似于Python中的字典。
  • Go语言的map是引用类型，不需要指明长度，是可以动态增长的。
  • 未显式初始化的map，默认使用零值nil进行初始化。
0x00 map声明及初始化

var test map[KeyType]ValueType

  • key 可以是string、int、float。数组、切片和结构体不能作为 key，但是指针和接口类型可以；
  • value可以是任意类型。
test := map[string]int{"mikasa": 1, "Aqua": 2, "Yuigahama": 3}   //声明并初始化
test := map[string]int{}  //不显式初始化时，使用nil进行初始化。

test := make(map[int]string,5)   //使用make函数构造map，分配内存。
test[1] = "Mikasa"               //新增一个元素
test[2] = "Yuigahama"

test := make(map[int][]string，[cap])  //map值为切片类型,cap为容量大小，不是必需的。
test[1] = []string{"Mikasa"}
test[2] = []string{"Yuigahama"}
0x01 映射特性

  • 将一个map赋值给另一个变量，与切片一样，是引用，它们的元素都引用于同一对象。

    func main() {
    m1 := map[string]int{"a": 1, "b": 2, "c": 3, "d": 4}
    m2 := m1

    m1["b"] = 22    //改变m1

    fmt.Println(m1)  //map[a:1 b:22 c:3 d:4]
    fmt.Println(m2)  //map[a:1 b:22 c:3 d:4]，m1改变导致m2也改变
}
0x02 映射操作

  • 遍历map

    test := map[int]string{1: "Mikasa", 2: "Yuigahama", 3: "Toka", 4: "Tosaki"}
for k, v := range test {
    fmt.Println(k, test[k], v)  //test[k]与v都可得到值。
}

  • 删除map中某个键值对：delete(map,key)

    test := map[int]string{1: "Mikasa", 2: "Yuigahama", 3: "Toka", 4: "Tosaki"}
delete(test, 1)   //删除test中key为1的键值对。

  • 判断key是否存在

    func main() {
    m1 := map[string]int{"a": 1, "b": 2, "c": 3, "d": 4}

    value, ok := m1["a"]
    if !ok {
        fmt.Println("key doesnt exist")
    } else {
        fmt.Println(value) //1
    }
}
0x03 sync.Map

  • 在并发情况下，同时对一个map进行读写会造成线程不安全，Go会检测到并报错。

  • sync.Map是一种线程安全的map,但是与原生的map有很大不同：

    • 无须初始化，直接声明即可。
    • sync.Map 不能使用 map 的方式进行取值和设置等操作，而是使用 sync.Map 的方法进行调用：
      • Store 表示存储
      • Load 表示获取
      • Delete 表示删除

  • 使用 Range 配合一个回调函数进行遍历操作，通过回调函数返回内部遍历出来的值。

func main() {
    var info sync.Map

    info.Store("name", "Mikasa") //存储键值
    info.Store("age", 1000)
    info.Store("gender", "female")
    info.Store("test", "test")

    value, ok := info.Load("test") //取值
    if !ok {
        fmt.Println("No key")
    } else {
        fmt.Println(value)
    }

    info.Delete("test") //删除

    //遍历键值
    info.Range(func(k, v interface{}) bool {
        fmt.Println(k, v)
        return true
    })
}

四、注意事项总结

0x00 引用类型要分配内存才可使用

  • 下面的代码会引发panic,在Go语言中对于引用类型的变量，我们在使用的时候不仅要声明它，还要为它分配内存空间，否则我们的值就没办法存储。

    func main() {
    var (
        s1 []int
        m1 map[string]int
    )

    s1[0] = 1      //error: index out of range
    m1["a"] = 1    //panic: assignment to entry in nil map
}

    • 引用类型分配内存的两种方式

      • 声明同时初始化会发生内存分配。

        func main() {

    s := []int{0, 1}
    m := map[int]string{0: "Aqua"}

    s[1] = 2
    m[1] = "Mikasa"

    fmt.Println(s) //[0 2]
    fmt.Println(m) //map[0:Aqua 1:Mikasa]
}

      • 若不想在声明时进行初始化，则需要使用make()函数分配内存，会使用对应的0值进行初始化。

        func main() {

    s := make([]int, 4, 8)
    m := make(map[int]string)

    fmt.Println(s) //[0 0 0 0]
    fmt.Println(m) //map[]
}

  • 值类型声明即可使用，因为值类型在声明的时候已经默认分配好了内存空间。

    func main() {
    var test [4]int

    test[1] = 2

    fmt.Println(test)  //[0 2 0 0]
}
