Go Package—net\http(Part I：Client)

net\http http Client
Golang

一、http.Client

net\http包提供Client结构体来实现HTTP客户端功能。

Transport

Transport是真正用于网络通信的，包括建立TCP连接、建立TLS连接、发送HTTP Request等。

通常Transport都实现了TCP链接缓存功能，因此应该重用一个Client，而不是每发一个HTTP Request都要创建一个Client。

Client是线程安全的，不用担心多个goroutine使用同一个Client会造成问题。

CheckRedirct

CheckRedirect用来处理重定向跳转的情况，在进行重定向前，该函数会被调用。

Jar

Jar用于处理Cookie，包括在发送HTTP Reuqest时插入对应的Cookie,以及存储更新HTTP Response返回的Set-Cookie的值。

Timeout

Timeout用于设置超时时间。

type Client struct {
    // Transport specifies the mechanism by which individual HTTP requests are made.
    // If nil, DefaultTransport is used.
    Transport RoundTripper

    // CheckRedirect specifies the policy for handling redirects.
    // If CheckRedirect is not nil, the client calls it before following an HTTP redirect. 
    // The arguments req and via are the upcoming request and the requests made already, oldest first. 
    // If CheckRedirect is nil, the Client uses its default policy,which is to stop after 10 consecutive requests.
    CheckRedirect func(req *Request, via []*Request) error

    // Jar specifies the cookie jar.
    // The Jar is used to insert relevant cookies into every
    // outbound Request and is updated with the cookie values
    // of every inbound Response.
    // The Jar is consulted for every redirect that the Client follows.
    // If Jar is nil, cookies are only sent if they are explicitly set on the Request.
    Jar CookieJar

    // Timeout specifies a time limit for requests made by this Client. 
    // The timeout includes connection time, any redirects, and reading the response body. 
    // The timer remains running after Get, Head, Post, or Do return and will interrupt reading of the Response.Body.
    // A Timeout of zero means no timeout.
    Timeout time.Duration
}

http.DefaultClient

http.DefaultClient是http包提供的默认的Client，http.Get()、http.Post()、http.PostForm()、http.Head()都是基于该默认结构体发送HTTP Request。

// DefaultClient is the default Client and is used by Get, Head, and Post.
var DefaultClient = &Client{}

二、RoundTripper

RoundTripper是一个接口，其RoundTrip()方法用于建立TCP链接、发送HTTP Request返回HTTP Response等。

// RoundTripper is an interface representing the ability to execute a
// single HTTP transaction, obtaining the Response for a given Request.
//
// A RoundTripper must be safe for concurrent use by multiple goroutines.
type RoundTripper interface {
    // RoundTrip executes a single HTTP transaction, returning
    // a Response for the provided Request.
    RoundTrip(*Request) (*Response, error)
}

http.Transport

http.Transport是http包提供的一个实现了RoundTripper接口的结构体。

我们可以使用该结构体定制化HTTP链接，如HTTP连接数、TLS、HTTP代理等,并在实例化Client时传入。。

http包同时也提供了默认的DefaultTransport，主要用于默认的DefaultClient使用。


// DefaultTransport is the default implementation of Transport and is used by DefaultClient. 
var DefaultTransport RoundTripper = &Transport{
    Proxy: ProxyFromEnvironment,
    DialContext: (&net.Dialer{
        Timeout:   30 * time.Second,
        KeepAlive: 30 * time.Second,
        DualStack: true,
    }).DialContext,
    ForceAttemptHTTP2:     true,
    MaxIdleConns:          100,
    IdleConnTimeout:       90 * time.Second,
    TLSHandshakeTimeout:   10 * time.Second,
    ExpectContinueTimeout: 1 * time.Second,
}

三、CookieJar

CookieJar主要用于存取Cookie，其内部使用嵌套的map来存储Cookie。

  • map[string]map[string]Cookie
    • 最外层map的key是URL字符串。
    • 内层map的key是id值（id由Domain+Path+Cookie的键名组成）。
    • 内层map的值是Cookie结构体，即一个Cookie键值对。

CookieJar是一个接口,声明了两个方法。

  • SetCookies()
    • 提供向CookieJar中存储Cookie的接口。
  • Cookies()
    • 提供从CookieJar中获取对应URL的Cookie键值对的接口。
// A CookieJar manages storage and use of cookies in HTTP requests.
//
// Implementations of CookieJar must be safe for concurrent use by multiple goroutines.
//
// The net/http/cookiejar package provides a CookieJar implementation.
type CookieJar interface {
    // SetCookies handles the receipt of the cookies in a reply for the
    // given URL.  It may or may not choose to save the cookies, depending
    // on the jar's policy and implementation.
    SetCookies(u *url.URL, cookies []*Cookie)

    // Cookies returns the cookies to send in a request for the given URL.
    // It is up to the implementation to honor the standard cookie use
    // restrictions such as in RFC 6265.
    Cookies(u *url.URL) []*Cookie
}

http.Cookie

使用Cookie结构体来表示一个Cookie键值对。

type Cookie struct {
    Name  string
    Value string

    Path       string    // optional
    Domain     string    // optional
    Expires    time.Time // optional
    RawExpires string    // for reading cookies only

    // MaxAge=0 means no 'Max-Age' attribute specified.
    // MaxAge<0 means delete cookie now, equivalently 'Max-Age: 0'
    // MaxAge>0 means Max-Age attribute present and given in seconds
    MaxAge   int
    Secure   bool
    HttpOnly bool
    SameSite SameSite
    Raw      string
    Unparsed []string // Raw text of unparsed attribute-value pairs
}

cookiejar.Jar

net/http/cookiejar包中Jar结构体提供了http.CookieJar的实现，并且提供cookiejar.New()实例化一个CookieJar。

// Jar implements the http.CookieJar interface from the net/http package.
type Jar struct {
    // 定义了
    psList PublicSuffixList

    // mu locks the remaining fields.
    mu sync.Mutex

    // entries is a set of entries, keyed by their eTLD+1 and subkeyed by
    // their name/domain/path.
    entries map[string]map[string]entry

    // nextSeqNum is the next sequence number assigned to a new cookie
    // created SetCookies.
    nextSeqNum uint64
}

// New returns a new cookie jar. A nil *Options is equivalent to a zero Options.
func New(o *Options) (*Jar, error) {
    jar := &Jar{
        entries: make(map[string]map[string]entry),
    }
    if o != nil {
        jar.psList = o.PublicSuffixList
    }
    return jar, nil
}

http.Client如何处理Cookie？

  • 实例化http.Client时，

    • 若没有传入CookieJar，即Jar==nil，那么发送的HTTP Request不会携带Cookie。

    • 若传入了CookieJar:

      • 在发送HTTP Request时，会调用CookieJar的Cookies()方法去查询是否有Cookie，查询到则添加到HTTP Request中。

      • 在接收到HTTP Response时，会调用CookieJar的SetCookies()方法将返回消息的Set-Cookie头部的Cookie存储到CookieJar中。

        func (c *Client) send(req *Request, deadline time.Time) (resp *Response, didTimeout func() bool, err error) {
  if c.Jar != nil {
      for _, cookie := range c.Jar.Cookies(req.URL) {  //从CookieJar中查询cookie。
          req.AddCookie(cookie)   //添加cookie到HTTP Request头部
      }
  }
  resp, didTimeout, err = send(req, c.transport(), deadline)
  if err != nil {
      return nil, didTimeout, err
  }
  if c.Jar != nil {
      if rc := resp.Cookies(); len(rc) > 0 {   //http.Response的Cookies()方法可以获取返回消息的Set-Cookie头部的值。
          c.Jar.SetCookies(req.URL, rc)   //保存Cookie到CookieJar
      }
  }
  return resp, nil, nil
}

四、Redirect

http.Client处理重定向的行为:

  • 发送HTTP Request，接收到HTTP Response后会调用redirectBehavior()判断是否需要重定向，函数根据HTTP响应码判断。
  • 若需要重定向则构建重定向请求，从HTTP Response的Location头部获取要重定向到的地址，设置重定向请求的Referer头部等。
  • 然后在发送重定向请求之前会调用在实例化http.Client时传入的checkRedirect函数。
    • 如果不想进行重定向，则在checkRedirect返回ErrUseLastResponse即可。
func (c *Client) do(req *Request) (retres *Response, reterr error) {
    ......
    for {
        // 重定向时才会执行该if代码段。
        if len(reqs) > 0 {
            loc := resp.Header.Get("Location")
            if loc == "" {
                resp.closeBody()
                return nil, uerr(fmt.Errorf("%d response missing Location header", resp.StatusCode))
            }
            u, err := req.URL.Parse(loc)
            if err != nil {
                resp.closeBody()
                return nil, uerr(fmt.Errorf("failed to parse Location header %q: %v", loc, err))
            }
            host := ""
            if req.Host != "" && req.Host != req.URL.Host {
                if u, _ := url.Parse(loc); u != nil && !u.IsAbs() {
                    host = req.Host
                }
            }
            ireq := reqs[0]
            req = &Request{
                Method:   redirectMethod,
                Response: resp,
                URL:      u,
                Header:   make(Header),
                Host:     host,
                Cancel:   ireq.Cancel,
                ctx:      ireq.ctx,
            }
            ......

            // Copy original headers before setting the Referer,in case the user set Referer on their first request.
            // If they really want to override, they can do it in their CheckRedirect func.
            copyHeaders(req)

            // 设置重定向请求的Referer头部
            if ref := refererForURL(reqs[len(reqs)-1].URL, req.URL); ref != "" {
                req.Header.Set("Referer", ref)

            // 在发送重定向HTTP Request之前，调用http.Client中的checkRedirect方法。
            err = c.checkRedirect(req, reqs)
            // 若不想进行重定向，只需要在checkRedirect函数中返回ErrUseLastResponse错误即可。
            if err == ErrUseLastResponse {
                return resp, nil

            ......
        }

  • redirectBehavior()

    响应码为301、302、303、307、308都会进行跳转。

    // redirectBehavior describes what should happen when the
// client encounters a 3xx status code from the server
func redirectBehavior(reqMethod string, resp *Response, ireq *Request) (redirectMethod string, shouldRedirect, includeBody bool) {
    switch resp.StatusCode {
    case 301, 302, 303:
        redirectMethod = reqMethod
        shouldRedirect = true
        includeBody = false

        if reqMethod != "GET" && reqMethod != "HEAD" {
            redirectMethod = "GET"
        }
    case 307, 308:
        redirectMethod = reqMethod
        shouldRedirect = true
        includeBody = true

        // Treat 307 and 308 specially, since they're new in Go 1.8, and they also require re-sending the request body.
        if resp.Header.Get("Location") == "" {
            shouldRedirect = false
            break
        }
        if ireq.GetBody == nil && ireq.outgoingLength() != 0 {
            shouldRedirect = false
        }
    }
    return redirectMethod, shouldRedirect, includeBody
}

五、http.Request

Request结构体用于存储http请求消息，同时用于客户端和服务端。

  • 在Client端，用于构造要发送的HTTP请求消息。（我们构造的Request结构体会被解析为HTTP request消息。）

  • 在Server端，用于接收客户端发来的HTTP请求（即服务器会把HTTP request解析到Request结构体中）。

    // A Request represents an HTTP request received by a server
// or to be sent by a client.
type Request struct {
    // 存储HTTP的请求方法。
    // 在Client端，不进行指定时默认使用GET。
    // Server端用于存储客户端的请求方法。
    Method string

    // 在Client端，URL结构体指明了要访问的服务器的URL信息。
    // 在Server端，保存解析了HTTP请求消息的请求行后的数据。请求的URI也会保存到RequestURI字段。
    URL *url.URL

    //对于Client端，以下三个字段都将被忽略。客户端总是使用HTTP/1.1 or HTTP/2.
    //对于Server端，用于获取客户端HTTP请求的协议版本。    
    Proto      string // "HTTP/1.0"
    ProtoMajor int    // 1
    ProtoMinor int    // 0

    // 对于Client端，用于指定请求消息的请求头。
    // 对于Server端，存储解析HTTP消息请求头得到的数据。注：Host请求头不会被解析存储到其中，而会被解析到Request.Host字段中。
        //    Host: example.com
        //    accept-encoding: gzip, deflate
        //    Accept-Language: en-us
        //    fOO: Bar
        //    foo: two
        //
        // then
        //
        //    Header = map[string][]string{
        //        "Accept-Encoding": {"gzip, deflate"},
        //        "Accept-Language": {"en-us"},
        //        "Foo": {"Bar", "two"},
        //    }
    Header Header

    //该字段代表HTTP请求消息的请求体。
    //对于Client，传入nil时，表示无请求体，例如GET请求就传入设为nil。POST等请求传入相应的请求体。客户端中Body的Close()方法由Transport负责调用。
    //对于Server，Body永远不为nil，但对于GET请求，读取Body会直接得到io.EOF。服务器端中Body的Close()方法由HTTP Server调用，而不是负责处理请求的ServeHTTP。
    Body io.ReadCloser

    //对于Server端,该字段无用。
    //对于Client端，函数用于获取Body的拷贝，用于需要多次使用一个Body时。
    GetBody func() (io.ReadCloser, error)

    //HTTP request Body的长度（字节数）
    ContentLength int64

    TransferEncoding []string
    Close bool

    //对于Client端，设置该字段会覆盖掉URL.Host的值。
    //对于Server端，该字段是Host头部的值。
    Host string

    //对于Client端，该字段无用。
    //对于Server端，Form用于存储解析form表单提交的数据（PATCH, POST, or PUT），另外还包括URL中的参数（URL.RawQuery）。需要先调用ParseForm函数进行解析，Form才有值。
    Form url.Values

    //对于Client端，该字段无用。
    //对于Server端，PostForm contains the parsed form data from PATCH, POST or PUT body parameters.也需要先调用ParseForm函数进行解析，才有值。
    PostForm url.Values

    // 对于Client端，该字段无用。
    // 对于Server端，该字段存储解析multipart form后的值，例如文件上传的。需要先调用ParseMultipartForm函数进行解析，才有值。
    MultipartForm *multipart.Form

    // Few HTTP clients, servers, or proxies support HTTP trailers.
    Trailer Header

    // This field is ignored by the HTTP client.
    RemoteAddr string

    // 对于Client端，不能设置该字段。It is an error to set this field in an HTTP client request.
    // 对于Server端，存储URI。
    RequestURI string

    // This field is ignored by the HTTP client.
    TLS *tls.ConnectionState

    //已废弃
    Cancel <-chan struct{}

    // This field is only populated during client redirects.
    Response *Response

    // ctx is either the client or server context. It should only
    // be modified via copying the whole Request using WithContext.
    // It is unexported to prevent people from using Context wrong
    // and mutating the contexts held by callers of the same request.
    ctx context.Context
}

六、http.Response

Response结构体用来表示一个HTTP Response消息。

// Response represents the response from an HTTP request.
//
// The Client and Transport return Responses from servers once
// the response headers have been received. The response body
// is streamed on demand as the Body field is read.
type Response struct {
    Status     string // e.g. "200 OK"
    StatusCode int    // e.g. 200
    Proto      string // e.g. "HTTP/1.0"
    ProtoMajor int    // e.g. 1
    ProtoMinor int    // e.g. 0

    // HTTP response消息头部，若多个头部的键相同，会被合并，值使用逗号分隔。
    Header Header

    // Body represents the response body.
    // The response body is streamed on demand as the Body field
    // is read. If the network connection fails or the server
    // terminates the response, Body.Read calls return an error.
    Body io.ReadCloser

    ContentLength int64
    TransferEncoding []string
    Close bool

    //是否解压缩
    Uncompressed bool

    Trailer Header

    // This is only populated for Client requests.
    Request *Request

    TLS *tls.ConnectionState
}
