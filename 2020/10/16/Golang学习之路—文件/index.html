一、打开/创建文件

  • 创建文件：os.Create()

    // Create方法用于创建文件（create）或清空文件内容(truncated). 
// 如果文件已存在，则是清空文件内容。如果文件不存在，则新建文件，新建的文件权限（perm）默认会被设置为0666。
// 如果Create方法成功调用，则会返回一个文件句柄（*File）,返回的文件句柄的操作模式（flag）为O_RDWR，即可读可写。
func Create(name string) (*File, error) {
    return OpenFile(name, O_RDWR|O_CREATE|O_TRUNC, 0666)
}

  • 打开文件：os.Open()

    // 传入文件名，返回一个文件句柄，返回的句柄操作模式为O_RDONLY，即只读。
func Open(name string) (*File, error) {
    return OpenFile(name, O_RDONLY, 0)
}

  • os.OpenFile()

    // 该方法是上面两个方法最终实际调用的方法，该函数是平台无关的。
// 方法体中的openFileNolog()才是在不同平台中有不同的实现，但我们不用去关注，因为OpenFile会自动调用到不同平台实现的openFileNolog()方法。
// flag是对函数返回的文件句柄的约束（例如，可读可写。。。）。
// perm是文件在操作系统文件系统中的权限，遵循Linux的权限设置设计（如0755）。
func OpenFile(name string, flag int, perm FileMode) (*File, error) {
    testlog.Open(name)
    f, err := openFileNolog(name, flag, perm)  
    if err != nil {
        return nil, err
    }
    f.appendMode = flag&O_APPEND != 0

    return f, nil
}

    • flag：flag理解为Go程序层面的权限，即Go程序对文件句柄的操作权限。

      // Flags to OpenFile wrapping those of the underlying system. Not all
// flags may be implemented on a given system.
const (
    // 打开的文件句柄必须设置为三者中的其中一个.
    O_RDONLY int = syscall.O_RDONLY // 以该标志打开的文件句柄只读。
    O_WRONLY int = syscall.O_WRONLY // 以该标志打开的文件句柄只能写入内容。
    O_RDWR   int = syscall.O_RDWR   // 以该标志打开的文件句柄可读可写。

    // 以下的用于搭配前三个使用.
    O_APPEND int = syscall.O_APPEND // 配合写权限，表示写入的内容是追加。（反例为O_TRUNC）
    O_CREATE int = syscall.O_CREAT  // 如果打开的文件不存在，则新建文件.
    O_EXCL   int = syscall.O_EXCL   // used with O_CREATE, file must not exist.
    O_SYNC   int = syscall.O_SYNC   // open for synchronous I/O.
    O_TRUNC  int = syscall.O_TRUNC  // 配合写权限，表示写入数据前会先清空文件内容。（反例为O_APPEND）
)

    • perm：perm是文件系统层面的权限。

      • 例：0755
        • 第一位是特殊权限位，0表示无特殊权限。
        • 755即rwxr-xr-x。

二、文件基本操作

Go使用os.File结构体表示文件操作符，os.Open()、os.Create()、os.OpenFile()返回的都是File结构体，结构体中封装了打开的文件的文件句柄。

事实上os.File是对os.file的封装，os.file才是真正用来表示文件句柄的结构体，但该结构体不同平台中也是不一样的，而os.File为用户屏蔽了不同平台的实现细节，是平台无关的。

// File represents an open file descriptor.
type File struct {
    *file // os specific
}

// Windows
// file is the real representation of *File.
// The extra level of indirection ensures that no clients of os
// can overwrite this data, which could cause the finalizer
// to close the wrong file descriptor.
type file struct {
    pfd        poll.FD
    name       string
    dirinfo    *dirInfo // nil unless directory being read
    appendMode bool     // whether file is opened for appending
}

// Unix
// file is the real representation of *File.
// The extra level of indirection ensures that no clients of os
// can overwrite this data, which could cause the finalizer
// to close the wrong file descriptor.
type file struct {
    pfd         poll.FD
    name        string
    dirinfo     *dirInfo // nil unless directory being read
    nonblock    bool     // whether we set nonblocking mode
    stdoutOrErr bool     // whether this is stdout or stderr
    appendMode  bool     // whether file is opened for appending
}
0x00 文件基本信息

  • func (file *File) Stat() (FileInfo, error)

    // A FileInfo describes a file and is returned by Stat and Lstat.
type FileInfo interface {
    Name() string       // base name of the file
    Size() int64        // length in bytes for regular files; system-dependent for others
    Mode() FileMode     // file mode bits
    ModTime() time.Time // modification time
    IsDir() bool        // abbreviation for Mode().IsDir()
    Sys() interface{}   // underlying data source (can return nil)
}

    • 例子

      func main() {
    fd, err := os.Open("./1.txt")
    if err != nil {
        fmt.Println(err)
    }

    fi, err := fd.Stat()
    if err != nil {
        fmt.Println(err)
    }

    fmt.Println(fi.Size())     //28
    fmt.Println(fi.Name())     //1.txt
    fmt.Println(fi.Mode())     //-rw-rw-rw-
    fmt.Println(fi.ModTime())  //2020-10-16 11:06:14.8511679 +0800 CST
    fmt.Println(fi.IsDir())    //false
}
0x01 读取文件

  • func (f *File) Read(b []byte) (n int, err error)

  • ioutil.ReadFile()：读取整个文件内容并返回。

    // ReadFile reads the file named by filename and returns the contents.
// A successful call returns err == nil, not err == EOF. Because ReadFile
// reads the whole file, it does not treat an EOF from Read as an error
// to be reported.
func ReadFile(filename string) ([]byte, error) {
    f, err := os.Open(filename)
    if err != nil {
        return nil, err
    }
    defer f.Close()
    var n int64 = bytes.MinRead

    if fi, err := f.Stat(); err == nil {
        if size := fi.Size() + bytes.MinRead; size > n {
            n = size
        }
    }
    return readAll(f, n)
}

    • 例子：

      func main() {
    result, err := ioutil.ReadFile("./test.txt")

    if err != nil {
        fmt.Println(err)
    }
    fmt.Println(string(result))
}

  • bufio.NewReader(fd)

    • func (b *Reader) Peek(n int) ([]byte, error) // read next n byte

    • func (b *Reader) ReadLine() (line []byte, isPrefix bool, err error) //读取数据直到遇到换行符。

    • func (b *Reader) ReadString(delim byte) (string, error) //读取数据直到遇到分隔符（delim）。

      func main() {
    f, err := os.Open("./test.txt") //打开文件，返回获得文件句柄及err
    defer f.Close()

    if err != nil {
        fmt.Println(err)
        return
    }

    reader := bufio.NewReader(f)    //封装*File

    var content []byte

    for {
        str, err := reader.ReadString('s')
        if err == io.EOF && len(str) == 0 {
            fmt.Println("文件读取完毕")
            break
        }
        if err != nil && err != io.EOF {
            fmt.Println(err)
        }
        fmt.Println(str)
        content = append(content, str...)
    }
    fmt.Println(string(content))
}
