一、接口（interface）定义

接口是一种类型。

接口是一种类型。

接口是一种类型。

  • 接口是一种约束类型，某个类型实现了某个接口，那么该类型就属于该接口类型。

    • 就可以用该接口类型的变量去接收实现了该接口的类型的数据。
    • 意味着一个接口类型的变量可以作为形参接收多个类型的数据。

  • 接口是一组方法的集合，接口只关注实现了什么方法，而不关心方法实现的具体细节。

  • 接口类型名即接口名，通常在名称后面加er表示接口。

  • 当接口名和方法名的首字母都是大写时，该方法可以被该方法所在包以外的地方访问。

  • 接口类型定义

    • 参数列表和返回值列表的变量名称可以省略，类型才是必须的，即可以只写类型。

      type 接口名 interface{
     方法名1( 参数1,参数2… ) (返回值1,返回值2)
     方法名2( 参数1,参数2… ) (返回值1,返回值2)
     …
}

二、接口实现

  • 实现接口的条件

    • 一个对象实现了接口的所有方法，那么就实现了这个接口。
    • 实现接口的类型的方法的签名必须要与接口的方法的签名一致。（函数名、参数、返回值）

  • 接口与类型的关系

    • 一个类型可以实现多个接口。

    • 一个接口可以被多个类型实现。

      type Car interface {
  run()
}

type Porsche struct {
  name string
}

type Benz struct {
  name string
}

func (p Porsche) run() {
  fmt.Printf("%s running...\n", p.name)
}

func (b Benz) run() {
  fmt.Printf("%s running...\n", b.name)
}

func drive(c Car) {
  c.run()
}

func main() {
  ps := Porsche{
      name: "Porsche",
  }
           
  bz := Benz{
      name: "Benz",
  }

  drive(ps) //Porsche running...
  drive(bz) //Benz running...
}

三、值接收器和指针接收器

  • 实现接口的方法时用值接收器则是结构体实现了接口。

    • 结构体及结构体指针都可以赋值给接口变量。

  • 实现接口函数时使用指针接收器是结构体指针实现了接口。

    • 只能使用结构体指针赋值给接口变量。

      type speaker interface {
  speaker()
}

type cat struct {
  name string
}

//指针接收器
func (c *cat) speaker() {
  fmt.Println("miao miao miao ~")
}

func main() {
  var s speaker

  c := cat{
      name: "feipei",
  }

  // s = c 报错，因为实现接口的不是cat，而是*cat，所以只能用结构体指针赋值给接口变量。
  s = &c
}

三、接口类型断言

  • 接口也是使用两个字长的数据结构。

    • 第一个字是当前接口存储的值的类型。

    • 第二个字是当前接口存储的对象。

      type tester interface {
  whatever()
}

type st1 struct {
  name string
}

type st2 struct {
  name string
}

func (s st1) whatever() {
  fmt.Println("st1")
}

func (s st2) whatever() {
  fmt.Println("st2")
}

func main() {
  s1 := st1{name: "s1"}
  s2 := st2{name: "s2"}

  var iface tester
  fmt.Printf("%T %v\n", iface, iface) //<nil> <nil>

  iface = s1
  fmt.Printf("%T %v\n", iface, iface) //main.st1 {s1}

  iface = s2
  fmt.Printf("%T %v\n", iface, iface) //main.st2 {s2}
}
类型断言

  • 类型断言即断言当前接口存储的值的类型。

    • x.(T) //断言x接口变量存储（指向）的值的类型为T。

    • 会返回两个参数：

      • 若断言成功返回的第一个参数是x接口变量指向的类型的值。
      • 第二个值是一个布尔值，若为true则表示断言成功，为false则表示断言失败。
      var x interface{}  //空接口可以接收任意类型的值。
i := 1
x = i  //此时x接口变量指向int类型的数据。

v, ok := x.(int) //断言，第一个参数接收值，第二个参数是判断是否为int的断言结果
if ok {
  fmt.Println(v)   //1
}

  • 若断言的类型是值类型，则返回的值是拷贝。断言引用类型返回的是引用。

    func main() {

    var a interface{}

    array1 := [3]int{1, 2, 3}
    slc := []int{4, 5, 6}

    //断言值类型
    a = array1
    v, ok := a.([3]int)
    if ok {
        fmt.Println(v) //[1 2 3]
        array1[0] = 111
        fmt.Println(v) //[1 2 3]。值未改变，因此值类型断言返回的值是拷贝。
    }

    //断言引用类型
    a = slc
    v1, ok := a.([]int)
    if ok {
        fmt.Println(v1) //[4 5 6]
        slc[0] = 111
        fmt.Println(v1) //[111 5 6]。值发生了改变，因此引用类型断言返回的是引用。
    }

}

四、空接口（interface{}）

空接口是没有定义任何方法的接口，因此所有类型都实现了空接口，因此空接口类型的变量可以接收存储任意类型的变量。

  • 空接口的主要应用：
    • 使用空接口作为函数形参，可以实现接收任意类型参数。 func test(a interface{}){}
    • 使用空接口作为map的值的类型，可以实现map的值为任意类型。 x := make( map[int]interface{} )
    • 空接口类型的chan可以实现传输任意类型数据。 ch := make(chan interface{})
